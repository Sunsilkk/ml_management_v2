## All Issues Accepted & Fixed in Round 5

### R5-ISSUE-1: Step 4 checkpoint state unsafe across attempts [ACCEPTED & FIXED]
Three changes: (1) Added `resume_source_checkpoint` column to job_steps — stores the source checkpoint path when resuming from a previous attempt (immutable after set, audit trail). (2) Added explicit new attempt reset rule in dispatch contract: when API creates new attempt_id, it atomically resets last_checkpoint_path=NULL, durable_checkpoint_iterations=[], and sets resume_source_checkpoint if applicable. (3) Updated column descriptions in schema to document reset behavior.

### R5-ISSUE-2: Log read semantics not attempt-aware [ACCEPTED & FIXED]
Three changes: (1) All public read APIs (metrics, evaluations, logs) now default to current_attempt_id with optional ?attempt_id= query param for historical debugging. (2) WebSocket live streaming explicitly resolves current_attempt_id from job and subscribes to attempt-scoped Redis channel. (3) Backfill reads from attempt-scoped MinIO paths.
